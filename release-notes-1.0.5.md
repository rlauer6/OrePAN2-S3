# OrePAN2::S3 Release Notes - v1.0.5

## Overview

This release focuses on build system modernization, bug fixes in the
bash wrapper and Perl module, a refactored `create-docs` command, and
a significant documentation overhaul. No breaking changes to existing
workflows.

---

## Bug Fixes

### Bash Wrapper (`orepan2-s3`)

- **`set -euo pipefail`** - corrected option order from the broken
  `set -oue pipefail` in the prior release.

- **`${INDEX_TEMPLATE:-}`** - fixed a shell variable expansion bug
  where `$INDEX_TEMPLATE{:-}` was passed literally, causing the
  template path to never be resolved from the environment.

- **`create_index` guarded with error handling** - both the `add` and
  `delete` commands now check the return value of `create_index` and
  emit an error message on failure rather than proceeding silently to
  cache invalidation.

- **`create-docs` invocation** - `--log-level` is now passed through
  when calling `orepan2-s3-index create-docs`, consistent with the
  `create` command. The progress message now includes the filename
  being processed.

### Module (`OrePAN2::S3`)

- **Wide character warning on upload** - `_upload_html` now encodes
  content as UTF-8 via `Encode::encode_utf8` before passing it to
  `Amazon::S3->add_key`, eliminating wide character warnings when
  uploading HTML containing non-ASCII characters.

- **Temp file leak in `cmd_create_index`** - the temporary
  `02packages.details.txt.gz` file downloaded during index creation
  was not being unlinked. The file is now explicitly removed after
  parsing.

- **Temp file leak in `cmd_show_orepan_index`** - same fix applied;
  the index temp file is now unlinked after the listing is assembled.

- **`fetch_orepan_index`** - simplified to always use `UNLINK =>
  $FALSE`, making callers explicitly responsible for cleanup.
  Previously the `$unlink` parameter defaulted to true, which caused
  `cmd_download_orepan_index` to attempt a `rename` on an
  already-deleted file.

## New Features

### `invalidate-index` Command

A new `invalidate-index` command has been added to the modulino
dispatch table. It is stubbed and not yet functional - it warns "Not
implemented yet" and exits cleanly. Full CloudFront invalidation
support is planned for v1.1.0.

### `cmd_create_docs` - Local Tarball Support

`create-docs` can now read a distribution directly from a local
tarball on disk rather than always fetching it from the DarkPAN URL.
If the path provided exists as a local file and does not look like a
URL or a `D/DU/` prefixed path, `Archive::Tar` is used to read it
directly. This avoids any caching issues with the remote mirror and
makes it easier to generate docs immediately after building a
distribution.

## Refactoring

### Command Sub Naming Convention

All command handler subs in `OrePAN2::S3` have been renamed with a
`cmd_` prefix to make the dispatch table self-documenting and avoid
any risk of collision with future accessor names generated by
`CLI::Simple`:

| Old name | New name |
|---|---|
| `create_index` | `cmd_create_index` |
| `show_orepan_index` | `cmd_show_orepan_index` |
| `create_docs` | `cmd_create_docs` |
| `download_orepan_index` | `cmd_download_orepan_index` |
| `upload_index` | `cmd_upload_index` |
| `invalidate_index` | `cmd_invalidate_index` |

### Boolean Constants

`$TRUE` and `$FALSE` local `Readonly` constants have been replaced
with `:booleans` imported from `CLI::Simple::Constants`, which also
provides `$SUCCESS`. The `Readonly` module is still present for
`$PACKAGE_INDEX` and `$DEFAULT_CONFIG`.

### `upload_html` Simplified

The `pod` flag and the `parse_pod()` code path have been removed from
`upload_html`. `DarkPAN::Utils::Docs->get_html` now handles both POD
and Markdown content uniformly. The `pod` argument is no longer
accepted.

### `orepan2-s3-index` Simplified

The modulino invocation script has been rewritten with a cleaner
`$INC`-based path lookup, removing the legacy `$EXTRA`, `$command`,
and `POSIXLY_CORRECT` cruft from the prior version.

### HTML Doctype

Both the embedded `__DATA__` index template and the `upload_html`
heredoc now use `<!DOCTYPE HTML>` (HTML5) rather than the HTML 4.01
Transitional doctype.

## Build System

The `Makefile` has been replaced with the generic version from
`CPAN::Maker::Bootstrapper`. Key improvements:

- Source files for scripts and the module are now maintained as `.in`
  templates; generated files are excluded from version control via
  `.gitignore`.
- `README.md` is generated automatically from the module POD via
  `pod2markdown`.
- Dependency scanning via `scandeps-static.pl` with
  preserve/skip/version override support in the `requires` file.
- `release-notes.mk` added - provides a `make release-notes` target
  that produces a diff tarball, file list, and unified diff against
  the last git tag.
- `buildspec.yml` corrected - project name, paths, URLs, and author
  email updated.

## Documentation

The POD in `OrePAN2::S3` and the generated `README.md` have been
substantially revised:

- **`NAME`** - now includes the required one-liner description for
  CPAN indexing.
- **`SYNOPSIS`** - new section with quick-start examples for both the
  bash wrapper and direct modulino invocation.
- **`DESCRIPTION`** - updated to accurately describe the module as a
  class/modulino rather than a plain script. Added a link to the TBC
  Development Group blog post on static S3 websites.
- **Commands** - `invalidate-index` added; bash-only commands (`add`,
  `delete`, `invalidate`) now documented separately under Notes with a
  clear explanation of which commands require the bash wrapper.
- **Options** - formatting and grammar corrected throughout.
- **Configuration** - several errors in the JSON examples fixed;
  `dump-template` example now correctly references `orepan2-s3-index`.
- **`SEE ALSO`** - expanded to include `DarkPAN::Utils`,
  `CLI::Simple`, and `Template`.
- **Author email** - standardized to `rlauer@treasurersbriefcase.com`
  throughout.
- Various typos corrected: "Commmands", "verison", "expresssion",
  "make want to", `02packages.details.text.gz`, `"files" {`.

## Known Limitations / Upcoming in v1.1.0

- `invalidate-index` is not yet implemented.
- Unit test coverage remains minimal (`use_ok` only). Expanded test
  coverage for `parse_index`, `fetch_config`, and `slurp_file` is
  planned.
